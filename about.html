    About This System

    A screening support tool, not a diagnosis

    OncoVision AI is designed to assist pathologists and clinicians by providing a second opinion on histopathology images. It classifies tissue as benign or malignant and shows a visual explanation of the regions that influenced that decision. It is not a replacement for expert clinical judgment.

    How it works

    The process is straightforward. You upload a histopathology image and the system returns a classification result alongside a heatmap showing which tissue regions the model focused on. The whole process takes a few seconds.

    01

    Upload an image

    Drag and drop or select a histopathology image in JPG or PNG format from the Analyze page.

    02

    Receive a classification

    The model returns a prediction of benign or malignant along with a confidence score indicating how certain the model is.

    03

    Review the explanation

    A Grad-CAM heatmap and a SHAP attribution map highlight the tissue regions that most influenced the prediction, so you can assess whether the model attended to clinically relevant areas.

    What the model can do

    The model was trained and evaluated on the BreakHis dataset, which contains histopathology images from 82 patients. The evaluation used a strict patient-level split, meaning the 1,148 test images came from 13 patients the model had never seen during training. This produces more honest performance estimates than the image-level splits used in much of the published literature.

    91.3%
    Malignant cases correctly identified (recall)
    78.4%
    Overall accuracy on unseen test images
    0.851
    AUC across all classification thresholds

    The system is configured to prioritise sensitivity, meaning it is more likely to flag a case as malignant when uncertain. Out of 647 malignant test cases, the model missed 56. Out of 501 benign test cases, 192 were incorrectly predicted as malignant. In a screening context, the cost of missing a malignancy is considered greater than the cost of an unnecessary follow-up.

    What it cannot do

    Being transparent about limitations is as important as reporting performance. This system has several constraints that any user should understand before relying on it.

    It is not a clinical diagnostic tool

    This system was developed as a research prototype and has not been validated for clinical use. All predictions must be reviewed by a qualified pathologist before any clinical decision is made.

    It was trained on a single dataset

    The model was trained and tested exclusively on the BreakHis dataset from one laboratory. Performance on images from different hospitals, scanners, or staining protocols is unknown and may differ.

    High false positive rate

    Around 38% of benign cases in the test set were predicted as malignant. This is a deliberate tradeoff to maximise sensitivity, but it means the system generates a significant number of false alarms that require clinical filtering.

    No uncertainty estimation

    The model does not currently provide a reliable measure of uncertainty for individual predictions. Some incorrect predictions were made with high confidence scores, which the confidence value alone would not flag.

    Images are resized during processing

    All images are resized to 224 by 224 pixels before classification. Fine-grained cellular detail present in the original image may be lost in this process, particularly at higher magnification levels.

    About the project

    This system was developed as a final year capstone project for a BSc in Computer Science at Ashesi University. Its goal is to assist clinicians by providing an interpretable breast cancer screening tool. The model classifies tissue as benign or malignant and highlights regions that influenced each prediction, helping users understand its reasoning.
